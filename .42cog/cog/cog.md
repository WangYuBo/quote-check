# 认知模型 (Cog)

> 本文件是 **本项目的实际认知模型**，不是模板。
> 写法规范参见同目录 `README.md`。

<cog>
本系统包括以下关键实体：
- 用户：系统使用者
  - 试用用户
  - 个人付费用户
  - 机构付费用户
  - 管理员
- 书稿：用户上传的、待校对的文稿
- 段落：书稿切分后的基本单位，也是"上下文"的载体
- 引文：书稿段落中被识别出的单条引用语句
- 参考文献：原始文献，用作核校引文的标准
- 校对任务：一次完整的校对作业（书稿 + 参考文献 + AI 校对流程）
- 校对结果：单条引文对某部参考文献的校对输出（三维度评分 + 置信度）
</cog>

<用户>
- 唯一编码：注册邮箱（MVP 单机模式下可为本地匿名 id）
- 常见分类：
  - 试用用户（按次或免费额度）
  - 个人付费用户（作者 / 研究生 / 自出版者）
  - 机构付费用户（出版社 / 编辑工作室，多席位）
  - 管理员
</用户>

<书稿>
- 唯一编码：task_id（上传时生成，形如 `20260417-001`）
- 常见分类：文史通俗读物；学术专著；古籍整理；学位论文；公众号 / 专栏文章
</书稿>

<段落>
- 唯一编码：`{task_id}-para-{段落顺序号}`，如 `20260417-001-para-042`
- 常见分类：按是否含引文——含引文段；纯叙述段；含脚注段
- 说明：判断"上下文相符性"时，取该段落 + 前后若干段作为上下文窗口
</段落>

<引文>
- 唯一编码：`{task_id}-quote-{顺序号}`，如 `20260417-001-quote-042`
- 常见分类：直接引用（带引号）；间接引用（化用）；标注引用（带脚注）
- 来源：对应 `origin/app/prompts/extract_quotes_prompt.txt` 的 AI 识别分类
</引文>

<参考文献>
- 唯一编码：`{slug}-{hash8}`，如 `lunyu-yangbojun-zhu-a1b2c3d4`
- 常见分类（按权威度）：
  - 原典（如《论语》原文）
  - 注本（如《论语集注》朱熹）
  - 现代译本（如《论语译注》杨伯峻）
  - 工具书（如《汉语大词典》）
</参考文献>

<校对任务>
- 唯一编码：task_id（与书稿共用同一 id）
- 常见分类（按状态）：待解析；解析中；校对中；已完成；失败
</校对任务>

<校对结果>
- 唯一编码：`{引文id}-{参考文献id}`
- 三维度判定（对应 `origin/app/prompts/verify_quote_prompt.txt`）：
  - **字词准确性**：通过 / 有异文 / 有错误
  - **解释一致性**：一致 / 部分一致 / 偏离
  - **上下文相符性**：恰当 / 存疑 / 断章取义
- 附加属性：置信度（0-1）、修改建议文本、原文定位（参考文献中的章节 / 段落）
</校对结果>

<rel>
- 用户-书稿：一对多（一个用户可上传多部书稿；每部书稿只属于一个上传用户）
- 用户-参考文献：多对多（参考文献可为个人私有或系统共享）
- 书稿-校对任务：一对多（同一书稿可触发多次校对——如更换参考文献重跑；MVP 当前为一对一）
- 书稿-段落：一对多（书稿切分为多个段落）
- 段落-引文：一对多（一个段落可能包含 0 条或多条引文；每条引文只归属一个段落）
- 书稿-参考文献：多对多（一部书稿可配多部参考文献；一部参考文献可被多部书稿引用）
- 引文-参考文献：多对多（一条引文可能有多个候选来源——异文 / 转引；一部参考文献可对应多条引文）
- 校对任务-校对结果：一对多（一次校对任务产出多条校对结果）
- 引文-校对结果：一对多（一条引文对多个候选参考文献产生多条校对结果）
- 参考文献-校对结果：一对多（一部参考文献可作为多条校对结果的来源）
</rel>

## 相对 v0 版本的主要修订

1. **关系修正**
   - `书稿-引文` 从 `多对多` 改为 `一对多`（引文是"一次引用行为"，绑定书稿上下文，不跨书稿复用）
   - `引文-参考文献` 从 `多对一` 改为 `多对多`（支持异文、转引、多版本对照）

2. **新增三个核心实体**
   - **段落**：上下文相符性判断的载体
   - **校对任务**：一次完整的校对作业及其状态
   - **校对结果**：三维度评分 + 置信度 + 修改建议，是系统的核心产出物

3. **分类与编码业务化**
   - 原 `[分类1][分类2][分类3]` 占位符全部填充为真实分类
   - 唯一编码从笼统的 `UUID` 改为有业务含义的结构化 id（task_id、slug+hash 等）
