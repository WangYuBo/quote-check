# 项目元信息 (Meta)

## 基本信息

**项目名称**: 文史类引用校对软件

**项目标语**: 智能核校文史类书稿中的应用语句是否符合原始文献、语句的解释是否准确、语句在当前上下文使用是否准确；

**版本**: 0.1.0

**创建日期**: 2026-04-17

**负责人**: yubo

## 项目描述
文史类书稿中有大量引用语句，人工核校需要大量机械时间。本项目旨在用AI来解决这一问题。计划由用户上传书稿、原始文献，本系统自动从书稿中找出所有引用语句、语句注释、语句上下文段落，然后与原始文献逐一核校，查验引用语句是否有字词错误、是否有解释错误、是否有误用，并给出AI对该核校的置信度。


## 目标用户

- 核心付费用户：高校/文史类出版社质检岗（B 端）
- 首发种子用户：人文社科研究生 + 自出版学者（C 端，冷启动友好）
- 辅助场景：出版社编辑日常校对（通过 B 端渗透）


## 核心价值主张

1. **三维度立体校对**：不止字词比对，同时核查**解释一致性**（作者诠释是否符合原文）和**上下文相符性**（是否断章取义）。对应 `origin/app/services/proofreader.py` 与 `origin/app/prompts/verify_quote_prompt.txt` 的三维度判断逻辑。

2. **可审计的证据链**：每条校对给出原文出处、AI 判断理由、置信度；支持 SSE 实时日志追踪 + Word/CSV 导出，编辑可秒级复核。

3. **成本透明可控**：10 万字书稿 500 条引用 ≈ ¥16.40（DeepSeek-V3.2），费用主要由引用条数决定，可预估可比价。这是相对 Grammarly/ProWritingAid（无文史场景）和 OpenAI 路线（成本高 3-5 倍）的关键差异点。

## 技术栈

### 当前实现（基于 `origin/` MVP）

**后端**：
- 语言：Python 3.10+
- 框架：FastAPI + uvicorn
- 并发：asyncio + 信号量（最多 8 条并发，限流自动重试 15/30/45/60 秒）
- 部署：单体服务（暂未上云）

**AI**：
- 主模型：DeepSeek-V3.2（via 硅基流动 API）
- 选择理由：古文理解接近 Claude，成本低 5-10 倍
- Prompt 文件化：`origin/app/prompts/extract_quotes_prompt.txt` + `verify_quote_prompt.txt`

**文件解析**：
- python-docx（DOCX）
- pdfplumber（PDF）
- ebooklib（EPUB）
- 原生 Python（Markdown / TXT）

**前端**：
- 原生 HTML / CSS / JS（`origin/templates/index.html` + `origin/static/`）
- SSE 实时进度推送

**数据**：
- 文件：本地 `origin/uploads/` 目录
- 暂无数据库

### 已知限制

- **原文自动截取前 20000 字符**：对《论语》《道德经》等短典籍没问题；对《资治通鉴》《四库全书》某部会漏——MVP 的隐性技术债，需在演进阶段通过 RAG/向量检索解决。

### 演进方向（待决策，非当前范围）

- **原典语料库接口层**：为后续接入古籍数据库（殆知阁、ctext.org、自建 pgvector）预留抽象
- **长文本处理升级**：突破 20000 字符限制，改用 RAG/向量检索
- **长任务队列**：书稿 30 万字+ 时从纯 asyncio 升级到 Celery/Arq
