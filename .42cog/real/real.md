# 现实约束 (Real)

> 本项目必须遵守的硬性限制。每一条都侧重"AI 不易想到、但在现实世界中会造成破坏"的事项。
> 其他工程约束与伦理边界参见 `notes/engineering-and-ethics-notes.md`。

## 必选约束（4 条）

<real required="true">

### 1. AI 只给提示，不得直接改稿；人工保留终审权
AI 输出的修改建议必须以"候选标注"的形式呈现，不得直接修改书稿原文。编辑/作者必须逐条人工确认后方可采纳。
**原因**：AI 幻觉 + 编辑盲信 = 错误修改被印成书，文史出版的学术声誉损失不可逆。

### 2. 每条校对结论必须附可追溯证据链
每条校对结果必须同时包含：
- 参考文献中的精确定位（书名 / 章节 / 段落）
- AI 判断理由（自然语言）
- 置信度（0-1，**不得由 AI 自评**，须用检索命中度 / 跨模型一致性等客观方式计算）

**原因**：没有证据链 = 编辑无法复核 = AI 结论无人敢用。这是 meta.md 核心价值主张"可审计证据链"的技术底线。

### 3. 用户上传内容受保密约束
- 书稿与参考文献不得用于任何模型训练
- 任务完成后 N 天内自动删除用户上传的原始文件（N 由用户协议约定，默认 7 天）
- 调用第三方 AI 服务（如硅基流动 / DeepSeek）必须在用户协议中明示数据流向

**原因**：用户上传大概率是未发表稿件。出版社采购的红线——一旦泄密事件发生，永久失去 B 端市场。

### 4. "版本异文" ≠ "引用错误"
校对判定必须区分两种情况，不得合并为单一"错误"标签：
- **版本异文**：引文与某权威版本不一致，但与另一权威版本一致（如《论语》皇侃本 vs 朱熹本）
- **引用错误**：引文与**所有**已知权威版本都不一致

**原因**：逐字比对会系统性误判异文为错误，诱导编辑"修正"本来就对的引用。这是文史校对的专业性红线。

</real>

## 可选约束（3 条）

<real required="false">

### 5. 参考文献版权责任在用户
- 用户上传前须声明自己有权使用该资料
- 系统默认**不**预置受版权保护的现代整理本（中华书局、上海古籍等校点本）
- 若未来接入第三方古籍数据库，须走正规授权渠道

**原因**：古籍原文公版，但现代校点 / 注释 / 译文受版权保护。预置受版权资料 = 侵权风险。

### 6. 单任务成本上限 + 二次确认
- 预估单次任务费用超过阈值（默认 ¥50）时，须向用户展示预估并二次确认
- 运行中费用超过预估 1.5 倍时自动暂停并通知用户

**原因**：按量计费下，100 万字 + 数千条引用可能烧到 ¥200+。对个人用户是暴击，对运营方是坏账。

### 7. 模型与 Prompt 版本锁定，旧报告不得悄悄替换
- 每份校对报告生成时必须记录：模型 ID + 模型版本 + prompt 版本 + 生成时间
- 模型升级或 prompt 迭代后，旧报告保留为历史快照，不得追溯修改
- 若需重新校对，生成新报告，旧报告并存

**原因**：学术场景下"稳定性 > 先进性"。编辑/作者事后复查时若结果对不上号，工具的可信度直接崩塌。

</real>
