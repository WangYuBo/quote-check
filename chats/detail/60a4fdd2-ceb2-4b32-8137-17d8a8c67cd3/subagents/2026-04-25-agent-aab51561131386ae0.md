### 7. API 路由认证保护 -- 存在但仅限于服务端

**所有** 数据 API 路由都在服务器端通过 `auth.api.getSession()` 检查会话。以下是带有完整会话检查的路由列表：

| API 路由 | 会话检查 | 所有权检查 |
|---|---|---|
| `POST /api/manuscripts` | 是 -- 401 如果未登录 | 否（在创建时关联 userId） |
| `POST /api/references` | 是 -- 401 如果未登录 | 否 |
| `GET /api/references` | 是 -- 401 如果未登录 | 是 -- 仅返回当前用户的引用 |
| `POST /api/tasks` | 是 -- 401 如果未登录 | 是 -- 403 如果 userId 不匹配 |
| `GET /api/tasks/[id]` | 是 -- 401 如果未登录 | 是 -- 403 如果 userId 不匹配 |
| `GET /api/tasks/history` | 是 -- 401 如果未登录 | 是 -- 仅返回当前用户的任务 |
| `GET /api/reports/[taskId]` | 是 -- 401 如果未登录 | 是 -- 403 如果 userId 不匹配 |
| `GET /api/reports/[taskId]/export` | 是 -- 401 如果未登录 | 是 -- 403 如果 userId 不匹配 |
| `GET /api/reports/[taskId]/export-docx` | 是 -- 401 如果未登录 | 是 -- 403 如果 userId 不匹配 |
| `GET /api/agreement` | 是 -- 401 如果未登录 | 否 |
| `POST /api/agreement` | 是 -- 401 如果未登录 | 否 |
| `GET /api/admin/audit` | 是 -- 401 如果未登录 | 是 -- 仅返回当前用户的审计日志 |
| `GET /api/admin/health` | **否** -- 公开的 | 否 |

通用模式：
```ts
const session = await auth.api.getSession({ headers: req.headers });

}
```

```ts
if (report.task.userId !== session.user.id) {
```

唯一进行认证检查的页面是 `app/reports/[taskId]/page.tsx`，它是一个**异步服务器组件**，调用 `auth.api.getSession()` 和 `headers()`。该页面是唯一在没有会话时优雅地显示替代 UI（"请先登录"）的页面 -- 但它也没有重定向到 `/login`，它只是显示一个静态的提示信息。

### 9. 观察到的关键缺口

1. **当未登录用户访问 `/upload` 时，他们可以到达该页面**，但所有 API 调用（上传稿件、引用、创建任务）都会失败，并显示 401。错误对用户不友好，并且不指示需要登录。

2. **当未登录用户访问 `/tasks/[id]` 时**，他们会看到轮询界面，但 `GET /api/tasks/[id]` 会返回 401，导致显示"无法获取任务状态"。

3. **当未登录用户访问 `/history` 时**，他们会看到空状态，但 `GET /api/tasks/history` 返回 401，导致显示"无法加载历史记录"。

4. **没有客户端会话观察者** -- 没有组件导入 `authClient` 来检查 `getSession()` 或监听会话状态（协议模态框是唯一使用 `authClient` 的，也仅用于登出）。

5. **`AgreementModal`**（在根布局中渲染）通过调用 `GET /api/agreement` 来检查协议接受状态，如果用户未登录则会失败（401），但错误被捕获并静默处理（`catch(() => { /* 网络失败不阻断 */ })`），因此它不会为未登录用户显示。

6. 目前未使用 **Better Auth 的 `auth.api.$fetch`**，也没有设置客户端 `fetch` 包装器来捕获 401 并重定向到登录页面。

### 10. 上下文中的登录流程

登录页面（`app/login/page.tsx`）：
- `POST /api/auth/sign-in/email` 用于登录，`POST /api/auth/sign-up/email` 用于注册。
- 成功登录后：`router.push('/upload')`。
- 成功注册后：切换到登录模式并显示"注册成功，请登录"。
- 没有调用 `authClient.getSession()` 来验证实际的会话状态。
- 登录页面本身没有重定向（如果用户已登录，不会自动发送到 `/upload`）。
