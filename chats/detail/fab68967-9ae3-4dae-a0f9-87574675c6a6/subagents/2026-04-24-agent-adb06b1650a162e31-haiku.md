## Summary: MAS-2 (参考为准绳) Requirements & Architecture

### **1. MAS-2 Affordances & Judgment Criteria**

**Core Philosophy**: System reports *facts* about quote-reference alignment, never judges "correctness." Editing authority stays with the human editor.

**Three Key Affordances**:
1. **Upload 1–10 reference documents** (perceive) — each marked with "role" (CANON/ANNOTATED/TRANSLATED/TOOL/OTHER)
2. **System matches quotes against uploaded references** (internal action) — ANY match = MATCH (editing authority principle)
3. **Three-state tags** (perceive):
   - ✅ **符合参考文献** (MATCH): At least one uploaded reference hit
   - ❌ **不符合参考文献** (NOT_MATCH): All references failed to match (neutral language, no "error/wrong")
   - ❓ **参考文献中未找到** (NOT_FOUND_IN_REF): Quote's source work doesn't match any uploaded reference

**Judge Criteria** (from spec §2.2):
- Neutral fact-reporting only ("matches / doesn't match" not "correct / incorrect")
- System never adjudicates editing decisions
- Evidence-based output: quote + closest reference paragraph + character-level diff
- Prompt must avoid 12 forbidden phrases (see §6 below)

### **2. Four-State Match Semantics**

**Exact Definition** (spec-product-requirements.md §A07 + spec-database-design.md §4.1):

| State | Enum Value | Business Meaning | API Field |
|-------|-----------|-----------------|-----------|
| **Full Match** | `MATCH` | Quote found in ≥1 uploaded reference | `matchStatus = 'MATCH'` |
| **Partial Match** | `PARTIAL_MATCH` | Quote found in M/N references (M < N); list hits | `matchStatus = 'PARTIAL_MATCH'` + `resultReferenceHit[]` |
| **No Match** | `NOT_MATCH` | Quote not found in ANY uploaded reference | `matchStatus = 'NOT_MATCH'` |
| **Source Not Uploaded** | `NOT_FOUND_IN_REF` | Quote's work (e.g., "史记") not in upload set | `matchStatus = 'NOT_FOUND_IN_REF'` |

**Enum Location**: `/lib/db/schema.ts` lines 50–55 (`matchStatusEnum`)  
**Constraint**: `verdict_text_accuracy.verdict` must be one of `['MATCH', 'VARIANT', 'MISMATCH', 'NOT_FOUND_IN_REF']` (verificationResult schema, lines 348–354)
