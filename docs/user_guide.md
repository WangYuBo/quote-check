## 1. 功能介绍

图书引用校对工具帮助作者、编辑在出版前快速核查书稿中的引用是否准确。

工具接收两份文档：

- **书稿（manuscript）**：待校对的书稿，其中包含引用他人著作的段落。
- **参考原文（source）**：被引用的原始文献，如《道德经》《论语》等经典原文。

系统自动完成以下工作：

1. 从书稿中识别并提取所有引用片段。
2. 将每条引用与参考原文逐一比对。
3. 从三个维度输出校对结论：**引用文字准确性**、**作者解释一致性**、**上下文相符性**。
4. 对存在问题的引用给出具体的修改建议。

适用场景包括学术专著、科普读物、文化类图书的引用核查，尤其适合大量引用古典文献的书稿。

## 2. 快速开始

### 2.1 环境要求

- Python 3.11 及以上
- Anthropic API Key（用于调用 Claude 模型）

### 2.2 安装依赖

```bash
# 进入项目目录
cd /path/to/proofreader

# 安装 Python 依赖
pip install -r requirements.txt
```

### 2.3 配置环境变量

在项目根目录创建 `.env` 文件，填入以下内容：

```dotenv
# 必填：Anthropic API 密钥
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxx

# 可选：使用的 Claude 模型（默认 claude-sonnet-4-6）
CLAUDE_MODEL=claude-sonnet-4-6

# 可选：单文件上传大小上限，单位 MB（默认 50）
MAX_UPLOAD_SIZE_MB=50

# 可选：开启调试模式（默认 false）
DEBUG=false
```

> 如果没有 `.env` 文件，也可以直接在终端中 `export ANTHROPIC_API_KEY=...` 设置环境变量。

### 2.4 启动服务

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

服务启动后，在浏览器中访问 `http://localhost:8000` 即可使用。

## 3. 操作流程

### 步骤一：进入首页

打开浏览器，访问 `http://localhost:8000`。页面中央显示两个文件上传区域，分别用于上传"书稿"和"参考原文"。

### 步骤二：上传书稿

点击"选择书稿文件"按钮，从本地选择待校对的书稿文件。支持 `.docx`、`.pdf`、`.md`、`.txt` 格式。书稿中应包含明确的引用段落（引号包裹或段落缩进标注均可被识别）。

### 步骤三：上传参考原文

点击"选择参考原文"按钮，上传与书稿引用来源对应的原始文献。原文格式同样支持 `.docx`、`.pdf`、`.md`、`.txt`。

> 注意：如果书稿引用了多个不同来源，当前版本每次只能上传一份参考原文。请分批上传，分别校对不同来源的引用。

### 步骤四：开始校对

点击"开始校对"按钮。系统将自动提取书稿中的引用，并逐条与参考原文对比。根据书稿篇幅和引用数量，校对通常在 30 秒至 3 分钟内完成。

### 步骤五：查看校对报告

校对完成后，页面自动跳转至报告页面，显示：

- **概览卡片**：引用总数、准确条数、存在问题条数。
- **逐条结果列表**：每条引用的三维度评分和具体说明。
- **下载按钮**：可将报告下载为 JSON 文件，留存备用。

## 4. 支持的文件格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| Word 文档 | `.docx` | 提取所有段落文本，忽略图片和表格内容 |
| PDF 文件 | `.pdf` | 逐页提取文本，扫描件（纯图片 PDF）无法识别 |
| Markdown | `.md` | 直接读取原始文本内容，保留换行结构 |
| 纯文本 | `.txt` | 支持 UTF-8 及 Latin-1 编码，自动探测 |

> 暂不支持 `.epub`、`.odt`、`.rtf` 等格式。如需处理这些格式，请先用相应工具转换为上述支持的格式后再上传。

## 5. 结果解读

每条引用的校对结果包含三个维度的评估：

### 5.1 引用文字准确性（text_accuracy）

检查书稿中的引用文字与参考原文是否逐字吻合。

- `is_accurate: true`：引用文字与原文完全一致，无错漏。
- `is_accurate: false`：存在文字差异，`issues` 字段会列出具体的差异内容，`original_text` 字段给出原文的正确写法。

常见问题类型：字词替换（如"惚恍"写成"恍惚"）、多字或漏字、标点符号差异。

### 5.2 作者解释一致性（interpretation_accuracy）

检查书稿作者对引用的解释、阐发是否与原文本义相符。

- `is_accurate: true`：作者的解释与原文语境和通行注疏基本一致。
- `is_accurate: false`：解释存在偏差或误读，`issues` 字段说明偏差所在，`suggestion` 字段给出修改建议。

### 5.3 上下文相符性（context_appropriateness）

检查引用是否符合其在书稿中所处段落的论述目的，即是否存在断章取义的问题。

- `is_appropriate: true`：引用与书稿上下文逻辑自洽，引用目的合理。
- `is_appropriate: false`：引用与上下文存在逻辑矛盾或语义不符，`issues` 字段给出具体说明。

### 5.4 综合建议（overall_suggestion）

在三个维度评估的基础上，AI 给出针对该条引用的整体修改建议，可直接参考用于修稿。

## 6. 常见问题 FAQ

**Q1：上传文件后提示"不支持的文件格式"，怎么处理？**

请确认文件扩展名为 `.docx`、`.pdf`、`.md` 或 `.txt` 之一。如果文件实际是 Word 文档但扩展名是 `.doc`（旧版 Word 格式），请用 Word 或 WPS 另存为 `.docx` 格式后重新上传。

---

**Q2：校对结果中某些引用显示"校对失败，无法判断"，原因是什么？**

这通常由以下原因导致：
- 网络问题导致 AI API 调用超时；
- ANTHROPIC_API_KEY 未正确配置或已过期；
- 单次请求的文本超出 AI 模型的上下文长度限制（系统已自动截取前 20000 字符，超长原文可能影响准确性）。

建议检查 API Key 是否有效，并在网络稳定时重试。

---

**Q3：书稿引用了多本文献，能一次全部校对吗？**

当前版本每次只支持一份参考原文。如果书稿引用了《道德经》和《论语》两本书，需要分两次上传：第一次上传书稿和《道德经》原文，第二次上传书稿和《论语》原文，分别获取校对报告后合并参考。

多来源一次性校对功能正在开发中。

---

**Q4：上传的 PDF 无法识别任何文字，是什么原因？**

PDF 分为两种类型：
- **文字 PDF**：直接包含可选中的文字，本工具可以正常解析。
- **扫描件 PDF**：本质是图片，没有可提取的文字层，本工具无法处理。

如果您的 PDF 是扫描件，请先使用 OCR 工具（如 Adobe Acrobat、ABBYY FineReader）将其转换为可搜索的文字 PDF，或导出为 `.txt` / `.docx` 文件后再上传。

---

**Q5：校对报告如何保存？能生成 Word 或 PDF 格式吗？**

报告页面提供"下载 JSON"按钮，可将完整结果保存为结构化数据文件，便于存档或二次处理。

如需 Word 或 PDF 格式的报告，可将 JSON 内容复制到文本编辑器整理后自行导出。原生导出 Word/PDF 格式的功能在后续版本中计划支持。
