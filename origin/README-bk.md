# 书稿引用校对系统

智能比对书稿引用与原文，精准识别文字差异、解释偏差与上下文问题。

## 功能特点

### 三维度校对
- **文字准确性**：逐字比对
- 引用与原文，识别文字、标点的增减改动
- **解释一致性**：核查作者解释是否准确反映原文意图，识别过度解读或曲解
- **上下文相符性**：评估引用是否恰当，检查断章取义或语境偏离

### 智能引用提取
- AI 自动识别书稿中的引用内容（引号引用、明确标注引用、间接引用）
- 支持分块并发提取，预过滤不含引用特征的段落提升效率
- 自动识别引用来源文献（如《论语》《道德经》等）

### 多格式支持
| 格式 | 书稿文件 | 参考原文 |
|------|----------|----------|
| DOCX | ✅ | ✅ |
| PDF | ✅ | ✅ |
| EPUB | ✅ | ✅ |
| Markdown | ✅ | ✅ |
| TXT | ✅ | ✅ |

### 多原文管理
- 同时上传多部参考原文（最多10部）
- 自动匹配引用与对应原文
- 缺失原文检测与提示

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置 API 密钥

创建 `.env` 文件：

```bash
SILICONFLOW_API_KEY=your_api_key_here
```

或使用环境变量：

```bash
export SILICONFLOW_API_KEY="your_api_key_here"
```

### 3. 启动服务

```bash
python -m app.main
```

或直接使用 uvicorn：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 4. 使用

浏览器访问 `http://localhost:8000`，上传书稿文件和参考原文即可开始校对。

## 工作流程

```
上传书稿 → 解析文件 → 提取引用 → 三维度校对 → 生成报告
    ↑                                              ↓
上传原文 ←─────────────────────────────── 导出 Word/CSV
```

1. **文件解析**：自动提取 DOCX/PDF/EPUB/Markdown/TXT 文本内容
2. **引用提取**：AI 识别并提取所有引用，记录段落位置、章节信息
3. **并发校对**：最多 8 条并发，自动重试限流请求
4. **结果展示**：统计概览、逐条详情、筛选查看、日志追踪
5. **报告导出**：支持 Word 表格或 CSV 格式导出

## API 接口

### 提交校对任务
```bash
curl -X POST "http://localhost:8000/api/proofread" \
  -F "manuscript=@书稿.docx" \
  -F "sources=@原文.pdf"
```

### 查询结果
```bash
curl "http://localhost:8000/api/result/{task_id}"
```

### SSE 实时进度
```bash
curl "http://localhost:8000/api/stream/{task_id}"
```

### 导出结果
```bash
# Word 格式
curl "http://localhost:8000/api/export/{task_id}?format=docx" \
  --output result.docx

# CSV 格式
curl "http://localhost:8000/api/export/{task_id}?format=csv" \
  --output result.csv
```

## 使用示例

**场景**：校对一本书稿中对《道德经》的引用

1. 准备书稿文件（包含引用内容的 DOCX/PDF）
2. 准备《道德经》原文（中华书局版等权威版本）
3. 上传两个文件，系统自动：
   - 提取书稿中的所有引用
   - 在原文中查找对应内容

   - 检查引用上下文是否合理
4. 查看报告，导出修改建议

## Token 消耗与费用估算

### 消耗构成

本工具使用 DeepSeek-V3.2 模型，调用分为两个阶段：

| 阶段 | 调用次数 | 单次输入 Tokens | 单次输出 Tokens |
|------|----------|-----------------|-----------------|
| 引用提取 | 书稿字数/1500（向上取整） | ~2,000-3,000 | ~500-1,000 |
| 引用校对 | 引用条数 | ~3,000-8,000 | ~500-800 |

> 注：引用校对的输入 tokens 主要取决于原文长度（自动截取前 20000 字符）

### 关键认知：调用次数 = 费用核心

本工具需要 **大量 API 调用**：
- 引用提取：书稿字数 ÷ 1500（向上取整）次调用
- 引用校对：每条引用 1 次调用

**10 万字书稿含 500 条引用 = 约 567 次 API 调用**

### 估算示例

以 **10 万字历史类书稿**、**500 条引用**、**10 本参考文献**为例：

**调用次数分解**：
| 阶段 | 调用次数 | 说明 |
|------|----------|------|
| 引用提取 | 67 次 | 10万字÷1500，并发执行 |
| 引用校对 | 500 次 | 每条引用单独调用 |
| **合计** | **567 次** | - |

**Token 计算**：
- 中文 1 汉字 ≈ 1-1.5 tokens（DeepSeek tokenizer）
- System Prompt（约 3000-4000 汉字）每次调用都需计入
- 原文截取 20000 汉字 ≈ 20000-30000 tokens

**引用提取阶段**（67 次）：
- 单次输入：System Prompt (~3500) + 书稿片段 (~2000) = ~5500 tokens
- 单次输出：~800 tokens
- 小计：输入 368,500 + 输出 53,600

**引用校对阶段**（500 次）：
- 单次输入：System Prompt (~4000) + 原文 (~25000) + 引用信息 (~1500) = ~30500 tokens
- 单次输出：~700 tokens
- 小计：输入 15,250,000 + 输出 350,000

**总计**：
- 总输入：~15,618,500 tokens
- 总输出：~403,600 tokens

### 费用参考（硅基流动）

| 模型 | 输入价格 | 输出价格 |
|------|----------|----------|
| DeepSeek-V3 | ¥0.001 / 1K tokens | ¥0.002 / 1K tokens |
| DeepSeek-R1 | ¥0.001-0.004 / 1K tokens | ¥0.002-0.016 / 1K tokens |

**上述示例估算费用**（按 DeepSeek-V3）：
- 输入：15,618,500 × ¥0.001/1000 ≈ **¥15.62**
- 输出：403,600 × ¥0.002/1000 ≈ **¥0.81**
- **总计约 ¥16.40**

> **重要**：原文大小不影响费用（自动截取前 20000 字符），只影响上传时间。费用主要由**引用数量**决定。

### 不同规模估算

| 书稿字数 | 预估引用数 | API 调用次数 | 预估费用 |
|----------|-----------|-------------|----------|
| 5 万字 | ~100 条 | ~134 次 | ~¥3.50 |
| 10 万字 | ~200 条 | ~267 次 | ~¥6.80 |
| 10 万字 | ~500 条 | ~567 次 | ~¥16.50 |
| 20 万字 | ~400 条 | ~534 次 | ~¥13.00 |

### 节省费用的建议

1. **精简原文**：只上传包含引用相关的章节，而非整本书
2. **分批处理**：大书稿分章节上传，避免一次性提取过多引用
3. **预过滤**：书稿中不含引用的段落会被自动过滤，减少无效调用

实际费用以硅基流动 API 账单为准，可在 [硅基流动控制台](https://cloud.siliconflow.cn/) 查看详细消耗。

## 注意事项

- 请使用权威版本的原文进行校对
- 原文文件越大，处理时间越长（自动截取前 20000 字符）
- 校对结果仅供参考，重要修改请以人工复核为准
- API 限流时会自动重试（最多 4 次，间隔 15/30/45/60 秒）
