## 致命级硬伤（系统根本无法工作）

### 1. 原文只取前2万字，100万字原文的98%被丢弃

`proofreader.py:31` 和 `:350`：

```python
SOURCE_TEXT_MAX_CHARS = 20000
truncated_source = source_text[:SOURCE_TEXT_MAX_CHARS]
```

这是**最致命的问题**。100万字原文只用前2万字，如果引用来自第3章以后的内容，**永远找不到对应原文**，会全部判为"原文中未找到对应内容"。

**当前没有任何检索机制**——不做全文搜索、不做向量检索、不做关键词定位，就是暴力截断。

### 2. 每次校对都把完整原文传给API，无定位能力

`verify_quote` 的 `user_message` 直接拼接截断后的原文。即使把 `SOURCE_TEXT_MAX_CHARS` 改大，DeepSeek-V3.2 的上下文窗口也容不下100万字。

**缺失的核心环节**：应该先用关键词/向量检索从100万字中定位到相关段落（几千字），再送去AI校对。

## 严重级硬伤（系统能跑但结果不可用/极慢）

### 3. 引用提取分块太小（1500字/块），15万字将产生海量API调用

`proofreader.py:37`：
```python
CHUNK_MAX_CHARS = 1500
```

15万字书稿经预过滤后假设保留30%（4.5万字），将产生 **~30个chunk**，每个chunk一次API调用。加上后续逐条校对，总API调用量可能达 **100-200次**。时间成本和费用都很高。

### 4. 内存任务存储，大文件并发时会OOM

`routes.py:63`：
```python
_tasks: dict[str, dict] = {}
```

100万字参考原文 ≈ 3MB纯文本，以 `source_texts: dict[str, str]` 形式常驻内存。多个任务同时运行时，加上段落列表、校对结果，内存会快速膨胀。服务重启则所有进行中任务丢失。

### 5. 文件解析把全部内容加载到内存

`file_parser.py` 的所有解析器都是全量读取。100万字PDF用 `pdfplumber` 全量解析会占用大量内存，且耗时很长。

### 6. 上传文件永不清理

`routes.py` 中 `_save_upload` 保存文件到 `./uploads`，**没有任何清理机制**。每次校对会保存书稿+原文副本，长期运行后磁盘会被占满。

---

## 中等级问题（功能缺陷/设计矛盾）

### 7. 架构文档与实际代码严重不符

`docs/architecture.md` 描述的模块结构（`quote_extractor.py`、`report_builder.py`、`proofread.py`）**实际都不存在**。文档说用 `Claude API` + `anthropic` 库，实际用的是 `硅基流动API` + `httpx`。`config.py` 中的环境变量名也与文档不一致。

### 8. 非docx格式的段落解析非常粗糙

`file_parser.py:273`：
```python
raw_paras = [p.strip() for p in plain_text.split("\n\n") if p.strip()]
```

15万字TXT/MD按双换行分段，可能产生几个巨大段落（如果原文没有空行）或上千个微小段落。没有智能分段逻辑。

### 9. 预过滤正则会漏检

`proofreader.py:46-48`：
```python
_QUOTE_LIKELY_RE = re.compile(
    r'[""「」《》]|引自|摘自|出自|所言|指出|认为|写道', re.UNICODE
)
```

大量引用形式未覆盖：`据...记载`、`...曰`、`...云`（古文常见）、`...有言`、脚注引用等。对于国学类书稿（如《道德经》注本），`曰` 和 `云` 是最常见的引用标志。

### 10. BackgroundTasks不适合长时间异步任务

FastAPI 的 `BackgroundTasks` 在同一个事件循环中运行。100+次API调用可能需要10-30分钟，期间会影响服务器响应其他请求。应该用 Celery 或独立的任务队列。

### 11. 前端只支持单文件参考原文上传（HTML层面）

`index.html:72` 的 `<input>` 没有 `multiple` 属性。虽然JS层面 `state.sourceFiles` 是数组，但HTML不支持一次选多个文件（只能反复点击逐个添加），体验很差。
